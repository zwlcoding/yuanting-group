## 工作流

### 项目流程
```
需求提出 → PRD撰写 → 需求评审 → TDD开发 → 测试验收 → 上线复盘
   ↑          ↓           ↓          ↓          ↓          ↓
  老板      摸总🐱     全员参与    混子💻    八阿哥🐛👑   亏总🎲
                              测试先行
```

### 各角色绝对权力

| 角色 | 权力范围 | 说明 |
|------|---------|------|
| 摸总 🐱 | 需求决策权 | PRD 写完后，不想改就不改 |
| 混子 💻 | 技术决策权 | 架构选型，说用啥就用啥 |
| 八阿哥 🐛👑 | 质量决策权 | 说不能上线，就不能上线 |
| 亏总 🎲 | 流程优化权 | 复盘总结，持续改进 |

### 核心价值观
- ✅ 不催进度 —— 了解进展和阻碍，但不施加压力
- ✅ 超配资源 —— 让团队无聊到只能创新
- ✅ 绝对权力 —— 专业的人做专业决策
- ✅ TDD 优先 —— 测试先行，质量第一
- ✅ 持续复盘 —— 每个项目结束必做总结优化
