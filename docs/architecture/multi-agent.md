# 渊渟集团多‑Agent 架构（v1）

> 目标：把“团队人设”落成可执行的多‑Agent协作系统，降低上下文污染、明确职责、提升协作效率。

---

## 二、系统结构

**单 Gateway，多 Agent，多 Workspace**

- **Gateway** 负责消息接入、路由、会话管理
- **Agent** 负责自身人格、记忆、规则、工具
- **Workspace** 为 Agent 的独立工作区（文件/记忆隔离）

```
openclaw
├── gateway (单进程)
└── agents
    ├── kuizong (统筹)
    ├── mozong  (产品)
    ├── hunzi   (开发)
    └── baage   (测试)
```

## 三、规则与记忆分层

### 3.1 文件即规则（每个 Agent Workspace）
- **SOUL.md**：人格/风格/底线
- **AGENTS.md**：启动流程/群聊规则/安全约束
- **IDENTITY.md**：身份卡片
- **USER.md**：用户画像（老板偏好）
- **MEMORY.md**：长期记忆
- **memory/YYYY-MM-DD.md**：每日流水
- **HEARTBEAT.md**：心跳清单

### 3.2 记忆分层策略
- **每日流水**：细碎、短期、高频
- **长期记忆**：稳定、可复用、需验证
- **语义召回**：按需加载，避免上下文膨胀

---

## 四、会话与协作机制

### 4.1 会话隔离
- 每个聊天窗口对应独立 session
- 禁止 Agent 间自动互 ping

**建议配置**：
```
{ "session": { "agentToAgent": { "maxPingPongTurns": 0 } } }
```

### 4.2 任务分派规则
- **需要以某角色身份对外发言** → `sessions_send` 发送到该角色 session
- **纯后台任务** → `sessions_spawn`
- **上下文过长** → 新开 thread（新 session）而非 spawn

## 五、模型与成本策略

| Agent | 推荐模型 | 说明 |
|------|----------|------|
| 亏总 | 中高性价比模型 | 统筹/路由/复盘为主 |
| 摸总 | 高质量语言模型 | 需求与表达质量优先 |
| 混子 | 代码模型 | 工具调用稳定、代码可靠 |
| 八阿哥 | 高质量语言模型 | 质检与评审更严谨 |

> 可按预算做 fallback：主模型故障自动切换次选模型。
